# 画像生成 & GIFアニメ作成タスク (4ポーズ → GIF)

## 絶対遵守
- **組み込み `image_gen` ツールを使う** (この環境は `~/.codex/auth.json` の `auth_mode: chatgpt`、CLI fallback `scripts/image_gen.py` は 401 で使えない、前回 401 確認済み)
- **1ポーズずつ 4回 image_gen を呼ぶ**。バッチ・並列・2x2スプライト一発生成は禁止 (ServerError で Pillow代替に落ちる)
- **エラー時に Pillow 図形描画フォールバック絶対禁止**。失敗したら停止 → 報告 → リトライ
- 既存の `site/assets/img/lure.png` を**参照画像として** image_gen の edit モードで渡し、同じキャラの別ポーズを生成する
- 透過は chroma-key (#00ff00) で生成、`$CODEX_HOME/skills/.system/imagegen/scripts/remove_chroma_key.py` で透過化

## 制作対象 (ルアー4ポーズ GIF アニメ)

参照画像: `site/assets/img/lure.png` (黄色のクランクベイト、ニコ目、ピンクほっぺ、クリーム腹、オレンジ背、ベージュリップ、フック)

### 4ポーズ (一つずつ、参照画像のキャラを完全保持して別動作で再描画)

1. `frame_1.png` — 体まっすぐ正面 (基準ポーズ。lure.png とほぼ同じ)
2. `frame_2.png` — 体が右側に少しくねる (尾びれが右、頭が左にスイング)
3. `frame_3.png` — 体まっすぐ (基準ポーズ近いが、口の表情や目の輝きを微妙に変えてリズム作る)
4. `frame_4.png` — 体が左側に少しくねる (尾びれが左、頭が右にスイング)

各フレーム共通仕様:
- サイズ 1024x1024
- 背景: 完全に均一な #00ff00 (chroma-key)
- ルアーは中央、サイズ・位置ほぼ揃える(GIFでズレないように)
- フラットカラー、黒輪郭なし、塊魂タッチ
- 影なし、テキストなし、ウォーターマークなし

### Codex 内部での image_gen 呼び出し例 (各フレーム個別)

参照画像を edit モードで渡し、ポーズだけ変える:

```
edit target: site/assets/img/lure.png
prompt: |
  Preserve the exact same yellow crankbait lure character from the reference image:
  same face, same nico-eyes, same pink cheeks, same body proportions, same color palette,
  same beige diving lip, same hooks, same orange back accent, same cream belly.
  Change ONLY the body posture to: <pose description>.
  Place the lure centered on a perfectly flat #00ff00 chroma-key background (1024x1024 canvas).
  No shadows, no text, no watermark. Sharp clean edges. Flat-color illustration, no black outlines.
constraints: same character identity, same colors, same style; only the pose/body curve changes
```

ポーズごとの description:
- frame_1: body straight, facing forward, tail relaxed
- frame_2: body bent slightly to the right, tail curved right (swimming wobble)
- frame_3: body straight again, slight tail shift compared to frame_1
- frame_4: body bent slightly to the left, tail curved left (opposite wobble)

## 後処理 (4フレーム完成後)

1. 各 chroma-key 入りフレームを `tmp/imagegen/frame_N_chromakey.png` に保存
2. `remove_chroma_key.py` で透過処理 → `tmp/imagegen/frame_N_alpha.png`
   ```bash
   python "$CODEX_HOME/skills/.system/imagegen/scripts/remove_chroma_key.py" \
     --input tmp/imagegen/frame_N_chromakey.png \
     --out tmp/imagegen/frame_N_alpha.png \
     --auto-key border --soft-matte \
     --transparent-threshold 12 --opaque-threshold 220 --despill
   ```
3. 4枚の透過PNGを Pillow で GIFアニメ化:
   ```python
   from PIL import Image
   frames = [Image.open(f"tmp/imagegen/frame_{i}_alpha.png") for i in range(1,5)]
   frames[0].save(
       "site/assets/img/lure-anim.gif",
       save_all=True,
       append_images=frames[1:],
       duration=200,  # 各フレーム200ms = 計800ms
       loop=0,
       disposal=2,
       transparency=0,
       optimize=False,
   )
   ```
4. 各フレーム透過PNGも `site/assets/img/lure-frame-1.png` 〜 `lure-frame-4.png` として保存(GIFと併用するため)

## 報告

- 4枚のフレーム生成 (成功/失敗、各ファイルパス)
- GIFアニメの最終パス・サイズ
- 中間ファイルは `tmp/imagegen/` に残してOK (デバッグ用)
- もし image_gen が ServerError や失敗を返したら**停止して報告**、絶対にPillowフォールバックしない
