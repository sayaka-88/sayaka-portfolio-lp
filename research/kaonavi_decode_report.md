# https://universe.kaonavi.jp/ 解析レポート（monokuro-decode）

作成: 2026-06-07 / 対象移植先: SAYAKA LP `site/`

## TL;DR（3行）
- **印象**: 「宇宙の旅」を縦スクロールで体験。ロケット/宇宙飛行士という"旅人"がシーンを移動し、固定背景の中を進む。全体が**ねっとり滑らかなスクロール**で繋がっている。
- **仕掛けの肝**: ①超ロング縦ページ(20×vh)で"スクロール尺"を確保 → ②`.s-fixed`固定ステージ + GSAP ScrollTrigger で進捗連動 → ③スムーズスクロール(イージング補間)で全要素がぬるっと動く → ④右の Index 宝石ナビをマーカーが下降。
- **移植難度**: 動きの"質感"だけなら**中**（Three.js本体は不要、GSAP+スムーズスクロールで質感再現可）。3D WebGLの完全再現は高。

## A. 構造マップ
- ライブラリ: **GSAP + ScrollTrigger**(164), **Three.js / WebGL**(123/242), Canvas。スムーズスクロールは独自実装（Lenis不使用 / `body.-after-scroll` クラス制御）。
- ビルド: Vite（`main-*.css` 339KB / `main-*.js` 754KB）。CSS @keyframes は `buttonPrimary` 1個のみ＝**動きはほぼ全部JS駆動**。
- セクション(`p-top-*`): `fv`(ヒーロー) → `op`(オープニング) → `story` → `unique`(VALUES円) → `invest` → `kpi` → `real` → `message` → `earth`(着地) → `last`。
- universal: 右 Index 宝石ナビ(Story/Invest/KPI/Real)、右下フローティングボタン群。
- ページ高 18,557px ≒ vh×20.5。

## A'. Visual 観察（スクショベース）
- **ヒーロー**: "kaonavi universe" ロゴ＋軌道リング＋惑星ドットがセンター固定。サブ「人的資本サイト」。
- **旅のナラティブ**: スクロールで大見出しが拡大上昇 → ロケットが左上へ斜め飛行 → 宇宙飛行士が旗を持って浮遊 → 惑星/月面に着地（砂地＋飛行士＋旗）。
- **VALUESハブ円**: 中心 "UNIQUE PERFORMER" の周りを5つの価値バブルが**周回**。スクロールで円が回転(`data-startang/rot`)し、進むと中身が「04 / 05」のリストに切り替わる。
- **ゾーン転換**: 白い液状の波形カットアウトが宇宙(濃紺)と地表(明色)の境界を作る＝ステージ転換の演出。
- **旅人＝移動体**: ロケット/宇宙飛行士が「スクロールに連れて移動する主役」。←★これが SAYAKA の「ルアー」に直結する核心。

## B. アニメーション機構カタログ
| 機構 | 駆動 | 値/挙動 |
|---|---|---|
| 擬似スクロール | JS(独自) | `.s-fixed{position:fixed}` の上で進捗を補間。全要素がイージングで遅延追従＝"ぬるっと" |
| ハブ円の回転 | ScrollTrigger | `data-startang`(50/59/135) 開始角、`data-rot`(75/105) 回転量。スクロール進捗→角度マッピング |
| 横ドリフト | ScrollTrigger | `data-x`(-0.4〜0.5) 要素ごとの横移動係数（パララックス） |
| 旅人移動 | ScrollTrigger | ロケット/飛行士がパスに沿って移動・スケール変化 |
| 背景パララックス | rAFループ | 星・粒子が層ごとに別速度 |
| Index マーカー下降 | ScrollTrigger | 進捗で右ナビのドットが Story→Invest→KPI→Real を下降 |

## C. JS trigger 機構
- 発火: `DOMContentLoaded` 後にスムーズスクロールループ起動 → `body` に `-after-scroll` 付与。
- 各セクションは `data-scroll=".p-top-xxx"` で自分のトリガー範囲を宣言、`data-group` で連動要素群を指定。
- **瞬間 `scrollTo` ではキャンバスが描画されない**＝描画がスムーズスクロールの補間値＋rAFに依存（実機確認済）。

## D. Stacking context 地図
- `.s-fixed`(固定ステージ, WebGL canvas) = 背面 / 各 `.js-section` テキスト = 前面 / Index ナビ・FABボタン = 最前面。
- 背景は濃紺(#2e3160 系) → 白波カットで明色ゾーンへ切替。

## E. craft の核心（なぜ気持ちいいか）
1. **スクロール＝移動の比喩**: 下にスクロール＝旅人が前へ進む、が一貫。
2. **慣性の効いたスムーズスクロール**: 全要素がワンテンポ遅れて追従し"重みと滑らかさ"を演出。
3. **固定ステージ上のレイヤー移動**: 背景固定・前景パララックスで奥行き。
4. **進捗インジケータ(Index宝石)**: 今どこにいるかを旅のメタファで可視化。

## F. craft翻訳プラン（SAYAKA LP 釣り版）

### 現状把握（重要）
SAYAKA LP `site/` には**既にキャスト演出が実装済み**:
- `.fishing-rod-wrap` 竿 / `.trail-lure` ルアー / `.cast-curve-path`(二次ベジェ糸)
- `playCast()`: ヒーロー抜けで竿を振る→ルアーが弧を描いて飛ぶ→着水splash/ripple
- スクロール進捗(`scrub:0.4`)でルアーが stage-surface→deep→space を**沈下**
- ゾーン構造: hero(about) → prelude → surface(海面) → undersea(works) → deep(深海) → space(宇宙) → dawn(voice) → landing(contact)
- 素材: `lure-new.png`, sky-/sea-/deep-/space-/land-* 完備

→ **ゼロ実装ではなく、既存キャストを kaonavi 級の"質感"に引き上げる**のが今回。

### 取り込むもの（優先度）
| 優先 | kaonavi の craft | 釣り版への翻訳 |
|---|---|---|
| **高** | 慣性スムーズスクロール（ぬるっと） | 全体にスムーズスクロール導入。ルアー沈下が一番効く |
| **高** | 旅人がスクロールでパス移動 | ルアーをスクロール進捗に**連続追従**させ、弧→沈下を一本のパスで滑らかに |
| **中** | 進捗インジケータ(Index宝石) | 右側に**水深ゲージ**(0m→深海)＋ルアーマーカー下降。LP各セクションを水深ポイントに |
| **中** | 背景パララックス(層別速度) | sea/deep/space の魚・泡・惑星素材を層別速度で漂わせる |
| **低** | ハブ円の回転 | （釣りには不要、見送り or Works を回遊グリッドに転用は将来） |

### 取り込まないもの
- **Three.js / WebGL 3D**: ポートフォリオLPに過剰・重量大(754KB)。GSAPで質感再現するため不要。
- ハブ円の周回回転（釣りの文脈に合わない）。
- アニメ的 narrative の作り込み（SAYAKAのトーンに合わせ抑制）。

### モノクロ流/SAYAKA流 再解釈
- 「宇宙の旅」→「**ルアーが空(宇宙)から放たれ、海面に着水し、深海へ沈む旅**」。素材がこの縦移動に完全対応済み。
- 旅人＝ルアー。進捗ゲージ＝釣り糸の水深。
