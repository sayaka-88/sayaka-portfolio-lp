# SAYAKA LP 下層ページ「フルポートフォリオ」仕様 v2

## 0. このページの役割

LP本体は「**入口・ストーリー体験**」、このページは「**詳細・網羅的なポートフォリオ**」。

- LP本体のワーク = ベストオブベスト（厳選代表作 × ジャンル別）
- このページ = **取引先名一覧 + 公開OK実績画像 + パスワード保護案件**を一気に見せる場
- LP本体の Stage 4（宇宙）からの「→ 詳しい取引先・実績はこちら」誘導の着地先

```
┌───────────────────────────────────────┐

├───────────────────────────────────────┤
```

## 2. 各セクションの詳細

### Section A：取引先一覧（上部）

**意図**：「これまでどんな方々と関わってきたか」を一目で見せる信頼担保。

**配置**
- ページ最上部、ヘッダー直下
- 静的な表示（スクロール演出なし、読みやすさ優先）
- 順不同に並べる（実績の偏りを意図的にぼかす）

**レイアウト**
- 見出し：「これまで関わった主な企業・案件（順不同・抜粋）」
- 社名はテキストのみ、グリッド or フラットな羅列
- ジャンルアイコンや業種ラベルは入れない（情報を絞る）
- 1行あたり3〜5社、画面全体で50〜100社入る想定

**コピー初稿**
- 見出し：「これまで関わった主な企業・案件（順不同・抜粋）」
- 注釈（小さく）：「すべての案件は掲載していません。守秘義務のあるものは下記からご覧いただけます」

**[要確認]**
- 表示する社名のリスト（SAYAKAから「正式リスト」共有予定）
- 一部の社名は「ジャンル / 業種」のラベル付きにするか、社名だけにするか

### Section B：実績デザイン画像（中部）

**意図**：取引先名だけだと味気ないので、画像で「実際の仕事の質」を網羅的に見せる。

**配置**
- Section A の下、ページの中央領域
- グリッドで画像を並べる（Pinterest風、3〜4列）
- クリックでモーダル拡大表示

**レイアウト案**
- 上部にジャンル切替タブ（全て / ロゴ / チラシ / パッケージ / グッズ / Web…）
- グリッド表示、画像メイン + 小さなキャプション（案件名 or ジャンル名）
- スクロール無限ロード or ページネーション（量に応じて）

**画像の選定基準**
- 公開してOKなもの限定
- LP本体のワークセクションに出した代表作はここにも入れる
- 加えて、LP本体には載せきれなかったが見せられる作品も全部入れる
- 棚卸しシート（[[SAYAKA_LP_client_triage.md]]）の「A」ラベルがついた案件群

**コピー初稿**
- 見出し：「制作実績」
- フィルター：「すべて / ロゴ / チラシ・フライヤー / ポスター / パッケージ / グッズ / 販促物・POP / SNSバナー / パンフレット / Web・LP」

**[要確認]**
- 画像の量（30点？100点？）
- フィルター方式（タブ / 検索 / なし）
- モーダル拡大時にキャプション・クライアント名を出すか

### Section C：パスワード保護案件（下部）

**意図**：契約上LPに載せられない案件を、商談時にだけ見せられるように。

**配置**
- ページ最下部、Section B の下
- パス入力前は **入力フォームと案内文だけ** 表示
- パス入力後にコンテンツがフェードインで表示

**レイアウト（入力前）**
- 見出し：「パスワードをお持ちの方へ」
- 説明文：「契約上LPでお見せできない案件は、こちらからご覧いただけます。パスワードはお問い合わせ・商談時にお渡ししています」
- パスワード入力欄 + 「見る」ボタン
- 不正解時：「パスワードが違います」のエラーメッセージ

**レイアウト（入力後）**
- 保護案件の画像がジャンル別 or 時系列でグリッド表示
- 見出し：「非公開実績」
- 各画像にクライアント名 or 案件名のキャプション

**コピー初稿**
- 入力前見出し：「パスワードをお持ちの方へ」
- 説明：「契約上、こちらのページでお見せできない実績がございます。ご相談・商談の際にパスワードをお渡ししています。お気軽にお問い合わせください」
- ボタン：「見る」
- エラー：「パスワードが正しくありません」

**[要確認]**
- パスワードの設定方針（固定 / 案件ごと / 期限あり）
- 表示する非公開案件の量と内容（LDH関連、Daiwa等）

## 3. パスワード保護の実装方針

**前提**：本格的なアクセス制御ではなく、「契約上の最低限の門」程度のクライアントサイド保護。

**実装方法（HTML/CSS/JSのみ）**
1. パスワードはJSコード内に SHA-256 ハッシュ化して保存
2. ユーザー入力をハッシュ化して比較
3. 正解なら `display: none` を解除して非公開案件の画像を表示

**セキュリティ上の限界**
- ブラウザでJSコードは見られるので、技術が分かる人は突破可能
- 画像のURLが直接知られれば、パスワードなしでもアクセス可能
- **「契約上、誰でも見られる場所には置いていない」という体裁を守るための仕組み**であって、本気のセキュリティではない

**強化したい場合の代替案**
- (a) Netlify / Cloudflare Pages の Password Protection 機能を使う（ホスティング側で保護）
- (b) サーバーサイドの認証を入れる（PHPやNode.jsが必要 → 技術スタックが変わる）

**確定（2026-05-19）**：**ホスティング側パスワード保護を採用**。HTML/CSS/JSのみ構成を維持しつつ、実用的なセキュリティが得られる。

**実装方針**
- ホスティング先候補：**Netlify**（Password Protection機能あり、無料プランで利用可）
- `/portfolio.html` の下層（パスワード保護案件パート）に対してアクセス制限をかける
- ただし Netlify の無料プランではサイト全体のパスワード保護のみ。Pro以上で個別ページ単位の保護が可能
- 代替案：Cloudflare Pages + Cloudflare Access（パスワードではなくメール認証だが無料で個別保護可）

**[要確認]**
- ホスティング先の最終確定（Netlify Pro / Cloudflare Pages / GitHub Pages等）
- Netlify無料プランで「サイト全体」保護にする場合は影響範囲を再考する必要あり

## 4. デザイン・トンマナ

LP本体と統一。

- 白基調 × 黒テキスト
- ゴールド差し色（明るめゴールド寄り）
- Noto Sans JP Light/Regular
- 余白広め
- フラットデザイン感

LP本体ほどの演出（GSAP・ScrollTrigger）は不要。**読みやすさ・網羅性**を優先。

シンプルなフェードイン程度のアニメは入れる。
